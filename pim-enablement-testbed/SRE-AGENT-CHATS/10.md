## What the tool is for

`get_user_group_memberships` resolves the biggest gap in the PIM validation workflow — it directly unblocks Rule **R004** (group membership check), which until now always had to emit `⚠ Cannot verify`.

It calls `GET /users/{id}/transitiveMemberOf` on Microsoft Graph and returns every group a user belongs to (including transitive / nested membership), filtered to objects of type `#microsoft.graph.group`.

## What this means for R004 validation

Now when the agent evaluates a PIM request, it can do a real check instead of `⚠ Cannot verify`:

```text
R004 check:
  1. Call get_user_group_memberships(requester_principal_id)
  2. Extract all group IDs from the response
  3. Compare against the approved on-call group IDs in validation-rules.yaml
  4. PASS if any match, FAIL if none match
```

Today's [validation-rules.yaml](../agent/validation-rules.yaml) still has placeholder group IDs:

```yaml
- "PLACEHOLDER-group-objectid-oncall-prod"
- "PLACEHOLDER-group-objectid-oncall-platform"
```

To make R004 fully functional, those placeholders need to be replaced with real group Object IDs:

- **Test tenant:** `a19dbeba-9826-4954-bde8-effef5dc7e85` (the group the test requester is in).
- **Zafin production tenant:** they would supply their on-call rotation group IDs (open ask **D1**).
